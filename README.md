# desktop-ski

Desktop-приложение на **PySide6** для работы с данными с **ESP32** в проекте «Акселерометр для откатки лыжных мазей скольжения».

Приложение предназначено для:
- подключения к микроконтроллеру ESP32 по USB;
- выгрузки сырых файлов с устройства;
- преобразования сырых TXT-логов в структурированные CSV;
- импорта уже подготовленных CSV-файлов;
- построения графиков ускорения, скорости, координат и траектории;
- экспорта данных в CSV;
- формирования PDF-отчёта по спуску.

## Назначение проекта

`desktop-ski` — это desktop-часть системы для сбора, загрузки, обработки и визуализации данных с датчиков, используемых в задаче анализа спуска и тестирования лыжных мазей скольжения.

Приложение объединяет в одном интерфейсе несколько этапов работы:
1. подключение к ESP32;
2. выгрузка файлов с микроконтроллера;
3. конвертация сырых данных в CSV;
4. импорт/экспорт данных;
5. визуализация сигналов и расчётных величин;
6. формирование отчёта в PDF.

## Основные возможности

### Работа с устройством
- Подключение к ESP32 по USB.
- Автоматический поиск доступного последовательного порта.
- Выгрузка файлов с микроконтроллера через `pyboard.py`.
- Обработка ошибок подключения и выгрузки через `QStatusBar`.

### Работа с данными
- Преобразование сырых TXT-логов в CSV.
- Импорт CSV-файлов в приложение.
- Экспорт текущего набора данных в CSV.
- Хранение загруженного датасета в памяти приложения.

### Визуализация
- Встроенный адаптивный Matplotlib-виджет.
- Построение набора графиков по загруженным данным.
- Автоматическая подстройка области графики под размер окна.

### Отчётность
- Экспорт PDF-отчёта по текущему спуску.
- Добавление шапки отчёта: когда создано и кем создано.
- Краткая таблица со сводкой по спуску.
- Встраивание графиков в PDF.

## Архитектура проекта

Проект организован по слоистой структуре.

### UI layer
Отвечает за окна, виджеты, кнопки, статус-бар и отображение графиков.

### Services layer
Содержит прикладную логику:
- подключение к ESP32;
- загрузка и импорт данных;
- подготовка данных для графиков;
- экспорт отчётов.

### Infrastructure layer
Содержит низкоуровневую работу:
- с `pyserial`;
- с `pyboard.py`;
- с файловой системой;
- с преобразованием сырых TXT в CSV.

### Core layer
Содержит:
- модели данных;
- прикладные исключения;
- структуры для графического слоя.

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/F4r3t/desktop-ski.git
cd desktop-ski
```

### 2. Создание виртуального окружения

**Windows (PowerShell):**
```bash
python -m venv .venv
.venv\Scripts\Activate.ps1
```

**Linux/macOS:**
```bash
python -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Запуск приложения

Из корня проекта:

```bash
python -m src.ski.app
```

Если такой способ запуска не используется в вашей структуре импорта, можно запускать напрямую файл приложения:

```bash
python src/ski/app.py
```

## Сценарий работы

Базовый сценарий приложения:

1. Запустить приложение.
2. Подключить ESP32 по USB.
3. Выполнить выгрузку данных с контроллера.
4. Автоматически или вручную получить CSV-файл.
5. Построить графики.
6. При необходимости:
    - экспортировать CSV;
    - сформировать PDF-отчёт.

Альтернативный сценарий:

1. Запустить приложение.
2. Импортировать уже готовый CSV-файл.
3. Построить графики.
4. Экспортировать CSV или PDF-отчёт.

## Формат входных данных

Приложение работает с CSV, содержащим сенсорные данные и временные метки.

### Основные поля

- `time` — исходное время из лога;
- `time_rel_ms` — время от начала эксперимента в миллисекундах;
- `time_rel_s` — время от начала эксперимента в секундах;
- `dt_ms` — шаг между соседними измерениями в миллисекундах;
- `gyro_x`, `gyro_y`, `gyro_z` — значения гироскопа;
- `accel_x`, `accel_y`, `accel_z` — значения акселерометра;
- `pressure` — давление.

### Конвертация из TXT

Если данные выгружены с ESP32 в виде TXT-лога, приложение использует конвертер, который:
- выделяет метаданные;
- выделяет блок измерений;
- рассчитывает относительное время;
- записывает CSV с основной таблицей данных;
- при необходимости сохраняет некорректные строки отдельно.

## Графики

После загрузки данных приложение строит следующие графики:

1. **Ускорение от времени**:
    - `a_x(t)`
    - `a_z(t)`

2. **Скорости от времени**:
    - `V_x(t)`
    - `V_z(t)`
    - `|V|(t)`

3. **Координата `x(t)`**

4. **Координата `z(t)`**

5. **Траектория `z(x)`** с раскраской по времени

Графики строятся во встроенном Matplotlib-виджете и автоматически адаптируются к размеру окна приложения.

## PDF-отчёт

Приложение формирует PDF-отчёт по текущему загруженному спуску.

### Структура отчёта

#### Шапка
- когда создано;
- кем создано;
- источник данных;
- имя файла.

#### Тело
- краткая таблица по спуску:
    - где был спуск;
    - когда был спуск;
    - температура;
    - влажность;
    - время спуска;
    - длина спуска;
    - максимальная скорость;
    - координаты геопозиций.

#### Графическая часть
- графики, построенные по текущему загруженному набору данных.

Если часть метаданных отсутствует в исходных данных или импортированном CSV, в отчёте для таких полей выводится `Не указано`.
